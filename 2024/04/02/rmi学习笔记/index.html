什么是RMI

Remote Method Invocation，也就是远程方法调用，通俗点解释，就是一台机器调用另一台机器上的java代码。比如我写了个Person类，里面有个printage方法。然后我朋友可以通过RMI在他的电脑上调用我写的Person.printage方法。当然这是非常浅薄的解释，实际上还会牵扯到RMI注册、RMI服务、RMI通信等等。。

既然是通信肯定要依赖通信协议，RMI依赖的通信协议是JRMP(Java Remote Message Protocol，Java 远程消息交换协议)，该协议为 Java 定制，要求服务端与客户端都为 Java 编写。

  • RMI分为如下三部分：

Server ———— 服务端：远程方法的提供者，并向Registry注册自身提供的服务
Client ———— 客户端：远程方法的消费者，从Registry获取远程方法的相关信息并且调用

Registry ———— 注册端；提供服务注册与服务获取。即 Server 端向 Registry 注册服务，比如地址、端口等一些信息，Client 端从 Registry 获取远程对象的一些信息，如地址、端口等，然后进行远程调用。

知识补充：

在计算机领域，Socket 是一种抽象概念，用于描述网络中两台计算机之间的通信端点。它实际上是一种编程接口（API），允许应用程序通过网络进行通信。

Socket 通常表示一个网络连接的端点，通过它可以在网络上发送和接收数据。在客户端-服务器模型中，客户端和服务器之间的通信通过套接字来实现。

套接字可以用于不同的通信方式，包括流套接字（Stream Socket）和数据报套接字（Datagram Socket）。

  • 流套接字（Stream Socket）：基于 TCP 协议的套接字，提供了可靠的、面向连接的通信。在流套接字上发送的数据是连续的字节流，保证了数据的顺序和可靠性，但可能会引入一定的延迟。
  • 数据报套接字（Datagram Socket）：基于 UDP 协议的套接字，提供了不可靠的、无连接的通信。数据报套接字发送的数据被分成一系列独立的数据包进行传输，不保证数据的顺序和可靠性，但具有较低的延迟。

通过套接字编程接口，程序员可以在应用层控制数据的发送和接收，从而实现网络通信。Socket 编程常用于网络应用开发，包括 Web 服务器、邮件客户端、即时通讯软件等。

调用过程：

Server部署：

  • Server向Registry注册远程对象，远程对象绑定在一个//hostL:port/objectname上，形成一个映射表（Service-Stub）。

Client调用：

  • Client向Registry通过RMI地址查询对应的远程引用（Stub）。这个远程引用包含了一个服务器主机名和端口号。

  • Client拿着Registry给它的远程引用，照着上面的服务器主机名、端口去连接提供服务的远程RMI服务器

  • Client传送给Server需要调用函数的输入参数，Server执行远程方法，并返回给Client执行结果。

RMI实现

服务端

1. 先编写一个远程接口，其中定义了一个 sayHello() 的方法

  • 此远程接口要求作用域为 public；
  • 继承 Remote 接口；
  • 让其中的接口方法抛出异常

2. 定义该接口的实现类 Impl

  • 实现远程接口
  • 继承 UnicastRemoteObject 类，用于生成 Stub（存根）和 Skeleton（骨架）。
  • 构造函数需要抛出一个RemoteException错误
  • 实现类中使用的对象必须都可序列化，即都继承java.io.Serializable

3. 注册远程对象

bind 方法是 Registry 接口的一个方法，用于将远程对象绑定到注册中心（Registry）。具体来说，bind 方法将远程对象关联到一个指定的名称，使得客户端可以通过这个名称来获取远程对象的引用。

bind 方法的参数如下：

  1. name（String 类型）：要绑定的名称，客户端将使用这个名称来查找远程对象。
  2. obj（Remote 类型）：要绑定的远程对象实例。
  • port 默认是 1099，不写会自动补上，其他端口必须写
  • bind 的绑定这里，只要和客户端去查找的 registry 一致即可。
